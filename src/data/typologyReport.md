# 憲法法庭審查結論類型學：構建、信度與資料集報告（v1.0）

版本：v1.0，2026-07-12 凍結。本報告對應機讀碼本 `data/materials/審查結論類型學.json`（`version: v1.0`）、人讀凍結碼本 `docs/審查結論類型學-凍結v1.0.md`、標籤真相檔 `data/processed/審查結論類型.json`、最終分布 `data/materials/類型學-最終分布.json`。三方數字須一致，任一方修訂須同步並升版重測信度。

## 一、摘要

為司法院大法官與憲法法庭全部 874 件行憲後裁判，建構一套六軸 37 類的審查結論類型學，在既有「合憲／違憲」粗軸之外記錄處分模式、失效技術、審查標的、對前解釋的處置、救濟指示與解釋權能六個獨立構念。標註以 `claude-opus-4-8` 雙盲兩獨立 pass 為主，分歧件經跨廠商第三票（GPT 家族 Codex）逐軸取多數，殘件人工裁定；860 件雙盲一致、9 件三票多數、5 件人工，最終不可自動判定率 0.11%（1 件）。凍結後以零重疊的 101 件新樣本重測，兩單選軸 Cohen's kappa 為 A=0.855、C=0.837，三多值軸 Jaccard 介於 0.827 至 0.911，F=0.970，全軸落於「幾乎完全一致」帶。每一標籤值附逐字引文（3392 條，機械驗證為主文或理由書真子串）。資料集可供時序分析、與立場表資料整合，並作為論文方法節的母本。

## 二、問題與構念

資料庫原有的「審查結論.結論」欄把裁判粗分為合憲、違憲、不受理等大類。這套粗分對「大法官做了什麼」的回答止於方向，抹掉了處分模式在台灣釋憲實務中的分殊。違憲宣告底下，主文可能諭知即時失效、定期失效、限期修正屆期失效，或對判例函釋宣告「不再援用」而不定失效日；合憲宣告底下，可能是單純合憲、讀入限縮條件的合憲性限縮，或附警告要求檢討改進的合憲；還有大量裁判根本不作合憲性判斷，只闡釋條文涵義或裁斷審判權歸屬。同標「違憲」的釋字第 291 號（即時失效）與釋字第 745 號（定期失效並課修法義務），在制度效果上判然有別，粗軸看不見這個差別。

六軸各鎖定一個構念。A 軸「處分模式」記錄合憲性判斷的基本性質（OUTCOME），單選互斥，是主軸。B 軸「違憲／位階處分技術」記錄違憲後的失效技術與結構標記（即時、定期、未定失效時點、排除適用、連帶失效、部分違憲、立法不作為型違憲），多值。C 軸「標的類型」記錄被審查或被解釋的對象是什麼（WHAT），單選，與 A 正交。D 軸「對前解釋的關係」記錄對大法官自身前解釋的補充、變更、維持、訂正、區別處置，多值，未觸及者空值。E 軸「救濟與後續指示」記錄主文附帶的修法義務、過渡規則、個案救濟、裁判憲法審查救濟、框架性立法要件、部分不受理款，多值。F 軸「解釋權能／法源」記錄行使憲法解釋抑或統一解釋權能，多值，一件可兼兩者。

類型學為粗軸補一組分析維度，保留而不取代既有欄位。審查強度（嚴格／中度／寬鬆）沿用資料庫既有「審查基準」欄；所依原則（法律保留、比例、平等、明確性、正當程序）視為既有欄位的子維度，不重造。六軸與這兩個既有欄位的分工是刻意的：類型學管處分的形態與客體，既有欄位管審查的強度與所據原則。逐軸 37 類的完整定義、判準、代表字號見凍結碼本第二節，本報告不重抄。

## 三、資料範圍

適用範圍為 874 件行憲後案：釋字第 1–813 號中的行憲後件、憲法法庭憲判，以及實體裁定（暫時處分裁定、程序裁定）。行憲前解釋與行憲前 5040 件排除在本類型學之外。排除的理由有二。行憲前解釋（1949 年前的院字、院解字系列）產生於訓政與行憲交界的憲政結構，其解釋權能、標的類型與行憲後的釋憲制度不共用同一組範疇，硬套六軸會製造名實不符的標籤。行憲前 5040 件為統字等大宗案件，性質偏個案法令適用而非憲法審查，與本類型學的構念不對齊。行憲前是否延伸為一套獨立、簡化的決定，留作開放問題（見第十節）。

理由書覆蓋是資料範圍內的一個事實界限。理由書檔涵蓋釋字第 80–813 號共 734 件全部有內容，缺理由書的 43 件恰為釋字第 1–79 號。早期大法官會議解釋本無獨立理由書，理由書慣例自釋字第 80 號起，這 43 件退回主文加爭點標註，其解釋文本身簡短自足，非解析漏失。

## 四、碼本

六軸各類別數為 A=9、B=7、C=8、D=5、E=6、F=2，合計 37 類。定義與判準逐字轉寫自機讀碼本，措辭以碼本為準。

| 軸 | 構念 | 單／多值 | 條件性 |
|---|---|---|---|
| A 處分模式 | 合憲性判斷的基本性質（OUTCOME） | 單選（恰取一值，互斥） | 每件必填 |
| B 違憲／位階處分技術 | 違憲後的失效技術與結構標記 | 多值 | 條件軸：A=A-P4 或 A-P9 適用；B-B4 例外，可獨立於 A |
| C 標的類型 | 被審查／被解釋對象（WHAT） | 單選（恰取一值） | 每件必填，與 A 正交 |
| D 對前解釋的關係 | 對大法官自身前解釋之處置 | 多值 | 條件軸：僅涉前解釋處置者填，未觸及者空值 |
| E 救濟與後續指示 | 主文附帶的救濟、修法、過渡、程序款項 | 多值 | 條件軸：有附帶指示者填 |
| F 解釋權能／法源 | 行使憲法解釋抑或統一解釋權能 | 多值 | 每件必填，一件可兼兩種權能 |

碼本以 append-only 方式演進，後續 draft 只補判準與決疑規則、不改既有 id 與定義。開發歷程中兩處迭代對終版測量有可證的效果，屬結果而非過程波折，記錄如下。

初版碼本經一次校準測試取得構念效度的第一道錨。將粗軸已知方向的合憲／違憲件盲標，A 軸方向相符率 92%；兩件不符經查為標註糾正了正則粗標之誤，實質相符高於帳面。這道測試確認六軸的類別劃分能對應真實法律範疇，不是空轉的標籤。

C 軸的主客體決疑規則是第二處迭代，其信度增益可測量。早期 C 軸在「闡釋法律用語涵義」（C-T5）與「前解釋作為被解釋客體」（C-T8）兩類之間界線不清：一件闡釋法律涵義而附帶對前解釋表態的裁判，究竟客體是該法律還是前解釋，兩位編碼員會分裂。draft-4 引入的規則把 C 定為「A 處分主文直接作用之規範」——附帶對前解釋的表態改記 D 軸，C-T8 僅限前解釋本身即為主要受審客體，一件涉數客體時取 A 直接處分者為 C，其餘由 B／D／E 多值軸承接。這條規則在凍結後的終版信度上把 C 軸 Cohen's kappa 從 0.770 拉到 0.837，直接回應了前一輪「C-T5／T8 邊界」的診斷。

C 與 F 的結構相依也在碼本中作了處理決定。874 件實測，以「各 C 類取眾數」預測 F，可導出率 95.0%（830/874），例外 44 件。高相關來自高頻大類（C-T1／T2／T3／T4 規範違憲審查幾乎全為 F-F1，C-T5 法律涵義幾乎全為 F-F2，合計 793 件、例外僅 9）。F 的獨立資訊集中在 C-T6／T7／T8 三類（機關權限、身分涵攝、前解釋處置，116 件，44 例外中 35 件落此），這三種案型既可能以憲法解釋權能作成、也可能以統一解釋權能作成，另有 5 件雙權能（F1+F2，如釋 177／218／305／751）任何單值導出規則都會抹掉。裁定保留 F 為獨立編碼軸，碼本明述其非正交（Cramér V≈0.93 係大類結構性伴生），下游分析使用 F 時應知此點。F 的 kappa 為六軸最高、編碼成本近零，砍軸的資訊損失換不到相稱的簡化。

## 五、標註程序

輸入規格為主文（cap 2200 字元）加理由書（cap 8000 字元）加爭點。判讀分工按軸的證據所在：A／B（處分性質與失效技術）以主文為錨，C／D／E（標的、對前解釋關係、救濟指示）優先讀理由書脈絡，這三軸的判準常只寫在理由書。固定 prompt 全文存 `docs/類型學-貼標prompt.md`，兩 pass 共用，凍結時同步鎖定；prompt 明令「現結論」欄禁止參考，依據須逐字引主文 ≥6 字，信心分三級。

標註採雙盲兩獨立 pass 加一致採納閘。`claude-opus-4-8` 兩個 pass 各自盲標，gate（`typology-tag-gate.mjs --agree`）要求 A、C 逐值相符、F 集合相符、依據為主文 NFKC 真子串，全部通過才採納。分歧件不硬收，併入 held 隊列。

全 874 件經四層收斂漏斗定案，各層件數依真相檔 `標註方式` 欄實測：

1. 雙盲一致採納：兩 pass 一致經 gate 者，860 件（`agent覆核`）在此層定案。
2. held 帶決疑規則重跑：兩 pass 分歧件併入 held，帶 draft-4 C 軸主客體決疑規則重跑，36 件收斂至 10。
3. 跨廠商三票 tie-break：held 殘件交 GPT 家族 Codex 獨立盲判當第三票，`typology-tiebreak.mjs` 逐軸取 opus1／opus2／Codex 的 2/3 多數，9 件得多數定案（`標註方式:三票多數`，信心中，附三票 provenance）。
4. 人工裁定：5 件經研究者裁定定案（釋 199／250／524／549／811），含全庫唯一真兩可件釋字第 250 號。

四層合計 860＋9＋5＝874，全庫定案。最終不可自動判定率為 0.11%——釋字第 250 號 C 軸三票全異（C-T2／C-T1／C-T4，標的本身跨層級，軍人轉服文職任用限制），人工裁定為 C-T1 法律，檔內保留裁定理由與三票原分歧供覆核翻案。這 1 件是編碼不可消除的模糊底線，不是分類器缺陷。

分歧即資料是這套程序的方法立場。兩模型對同批 kappa 為 0.82 而非 1.0，分歧本身證明有判斷空間；真正的法律邊界案（C-T5／T8、A-P4／P5）會讓任何兩位稱職編碼員分裂，那是現象本身模糊。追求 kappa=1.0 是錯的目標：0.99 反而可疑，要嘛任務太易、要嘛碼本硬壓了真實模糊。有紀錄的難案殘餘比假裝全對健康。

## 六、證據層

原始設計每件僅一條「依據」，作為 A 軸的反幻覺錨。這在多值軸留下可審計性缺口：174 件有兩個以上多值標籤共用一條引文，B／C／D／E／F 各標籤無逐值憑證。修法是一次證據增補 pass——凍結已驗證標籤不動，opus 單 pass 只對每個既有標籤找支持引文，寫入新欄 `依據s`＝{軸:值 → {引文, 來源:主文｜理由書}}，鍵集必須恰等於該件標籤的全部值。單 pass 不 churn 標籤，故不擾動 kappa。

覆蓋為 874/874 全覆蓋，3392 條逐字引文、12 條推定（0.35%）。推定通道供缺席界定值使用：無正面引文可引者記 {推定:理由}，作為防編造的例外通道。12 條推定集中兩型——B-B3「未定失效時點」以「主文無失效款」這一文本缺席界定，無正面引文可引；早期釋字 F-F1 主文無「憲法」字樣且無理由書。凍結時裁定容許推定作為缺席界定值的正式憑證通道。

機械驗真由 `typology-evidence-merge.mjs` 逐件執行：檢查鍵集相等、引文 NFKC 真子串於宣告來源、引文長度 ≥6 字，逐件全過才併。這道驗證固定進 `validate-processed.mjs` 的「依據s 逐值憑證」檢查（缺件、鍵集不符、引文非真子串皆 fail 級，推定僅計數），隨資料庫每次驗收重跑；引文對照 app 投影層主文（含官方文字勘誤）與理由書全文檔。機械層保證的是真子串，引文「支持標籤」的語意貼合度靠標註判斷加抽驗。

## 七、信度、效度與穩健性

編碼員間信度以兩異質家族模型的獨立盲標估計。opus 與 Codex（GPT 家族）各自對同一套六軸、同一碼本、同一輸入盲標，Codex 不見 opus 標籤。跨廠商設計排除了「兩 pass 同為 opus，一致率只證模型自我再現」的假象，兩個不同語料訓練出的模型獨立收斂，同時緩解預訓練污染的同錯風險。

| 軸 | 統計 | 84 件（凍結前） | 101 件（終版） |
|---|---|---|---|
| A 處分模式 | Cohen's kappa | 0.824（原始一致 86%） | 0.855（89%） |
| C 標的類型 | Cohen's kappa | 0.770（82%） | 0.837（87%） |
| B 違憲技術 | 平均 Jaccard／集合全等 | 0.904／85% | 0.911／89% |
| D 對前解釋 | 平均 Jaccard／集合全等 | 0.905／90% | 0.911／91% |
| E 救濟後續 | 平均 Jaccard／集合全等 | 0.905／89% | 0.827／82% |
| F 解釋權能 | 主權能單選化 kappa | 0.962（99%） | 0.917（Jaccard 0.970） |

判準：kappa≥0.8 幾乎完全一致，0.67–0.8 可發表門檻，0.6 以下需精修。終版 101 件樣本與 84 件樣本零重疊（年代×A 大類 12 層，種子決定性），全軸落於幾乎完全一致帶。C 自 0.770 升至 0.837 是 draft-4 決疑規則的實測增益。終版 A／C 分歧 19 件散於 P4/P2、P4/P3、P5/P6、T1/T2、T6/T8 等邊界對，無單點聚集，殘餘為本質模糊而非碼本缺陷。計算命令 `node engineering/scripts/typology-kappa.mjs data/materials/信度樣本2-opus標.json data/materials/信度樣本2-codex標.json`。

污染測試否證了「標籤靠對名案的訓練記憶」的假說。30 件（名案 12＋分層隨機 18）遮蔽字號、日期與文內自我引用後，opus 單 pass 盲標比對凍結標籤。名案層 A／C／F 一致率均 1.00（B Jaccard 0.917、E 0.875），隨機層 A 0.833、C／F 1.00。假設若成立，遮蔽應使名案層劣化最多；實測名案層不低於隨機層，標籤由文本驅動。隨機層 A 的 3/18 分歧與名案 D Jaccard 0.667 屬單 pass 對照多票收斂標籤的已知邊界變異——凍結標籤經雙盲加 tie-break，單 pass 本就略低。測試的限制在於文本內容本身（同婚、死刑等議題）對知名案件仍可識別，本測試測的是身分字串移除後的標籤穩定性，非完全匿名化。

重現性標準是信度範圍內的重現，不是逐位元重現。他人以凍結碼本、prompt 與鎖定模型版本（claude-opus-4-8）重編一樣本，一致度應達本報告的 kappa。LLM 即使溫度 0 也非完全確定、模型會被更新，追求逐件確定性重現對內容分析是錯的目標。可重現包等於凍結碼本加 prompt 披露加模型版本鎖加 kappa 加 gold 資料集。

## 八、結果：資料集面貌

以下只陳述分布，不作因果宣稱。分布數字以 `類型學-最終分布.json` 為準。

A 軸處分模式三分天下：A-P4 違憲宣告 251 件（28.7%）、A-P5 純解釋 247 件（28.3%）、A-P1 單純合憲 240 件（27.5%），三者近乎均分全庫八成；其餘為 A-P3 合憲附警告 60、A-P2 合憲性限縮 31、A-P6 機關權限歸屬 28、A-P7 程序性／暫時處分 8、A-P9 合法性審查 5、A-P8 不受理 4。純解釋佔比之高，反映大法官會議時代大量案件止於闡釋涵義或裁斷審判權，未進入合憲性審查。

B 軸失效技術（261 件有值）以 B-B4 排除適用 100 件最大宗，反映台灣釋憲對判例、函釋慣用「不再援用」而非「失其效力」的處分傳統；其次 B-B6 部分違憲 94、B-B2 定期失效 80、B-B3 未定失效時點 52、B-B1 即時失效 38、B-B7 立法不作為型違憲 24、B-B5 連帶失效 3。

C 軸標的類型以規範審查為主：C-T1 法律 315、C-T2 命令函釋 222 兩類佔全庫六成以上；C-T5 法律涵義 135、C-T4 憲法條文／機關權限 61、C-T3 判例決議 60；C-T6 審判權歸屬、C-T7 身分涵攝、C-T8 前解釋各 27，是三類低頻但編碼最難的標的。

D 軸（137 件有值）勾勒大法官對自身前解釋的處置網絡：D-D3 維持／援用 52、D-D1 補充 51 為兩大宗，D-D2 變更 20、D-D5 區別不予適用 10、D-D4 訂正澄清 8。變更前解釋的 20 件是研究釋憲見解演變的直接入口。

E 軸（170 件有值）以 E-E1 課予修法義務 95 為主，E-E2 過渡填補 51、E-E6 部分不受理款 48、E-E3 個案救濟 34、E-E4 裁判憲法審查救濟 13、E-E5 框架性立法要件 4。E-E4 專屬新制裁判憲法審查，其 13 件標記出 2022 年憲訴法施行後的新處分類型。

F 軸憲法解釋與統一解釋為 702：177，約 8:2；5 件雙權能兼行兩種解釋權，故 F 值合計 879。

跨軸結構有兩項聲明。C 與 F 高度相依（Cramér V≈0.93）係大類結構性伴生，F 的獨立資訊集中於 C-T6／T7／T8 與雙權能件，非正交軸。A 與 C 不冗餘（A×C 關聯約 0.45），OUTCOME 與 WHAT 是可分的兩個構念，這由 kappa 的可分性佐證。

## 九、限制

輸入截斷是第一項限制。理由書 cap 8000、主文 cap 2200，超長現代判決（如 113 憲判 8 死刑案、113 憲判 9 國會調查案）截斷處後的 D／E 訊號無法核實，標註未臆填，結果偏保守。

D 軸保守偏漏。D／E 定點補跑 103 候選中 86 件兩 pass 交集為空、確屬路過引用，補跑同時是驗證；截斷加保守判準使 D 可能有低估。缺席界定值靠推定：B-B3 等以文本缺席界定的值無正面引文，其憑證為推定（12 條之主體）。

編碼員是 LLM，這需要披露。標註由 `claude-opus-4-8` 與 GPT 家族 Codex 作成，非人類專家；信度為模型間 kappa 而非 human-agent kappa，構念效度靠校準測試（粗軸方向相符 92%）與污染測試佐證，仍非人工錨。

意見書刻意排除。協同與不同意見書按定義非判決本身，處分在主文加多數理由書；餵意見書會把少數見解污染進處分編碼。意見書歸另案的解釋方法分類器與立場表／表決分析，不納入本分類器。早期釋字（1–79 號）無獨立理由書，這 43 件退回主文加爭點，其 D／E 訊號本就稀薄。

## 十、後續研究接口

三條接口已就位。時序與屆期分析可接 A 軸分布隨任命時代的變化，檢驗「分殊化等於審議專業化，抑或任命政治化」的問題意識；此層需置換檢定（法官數少）、case-mix 控制、2022 年憲訴法斷點與受理守門的選擇偏誤處理，必須排在信效度層之後，標籤未校準前跑因果等於 garbage-in。與立場表資料的整合可把處分類型接上表決結構與意見書陣營，測度共識與爭議程度。行憲前 5040 件是否延伸類型學為開放決定，建議僅取 C 軸規則式標註或延後，不硬套六軸。
